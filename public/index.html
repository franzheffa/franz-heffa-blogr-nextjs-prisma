Buttertech – passerelle d’agents IA pour l’entreprise

Nous construisons une plateforme d’agents IA spécialisés (recherche, RAG, automatisation) orchestrés derrière une API Gateway sur Google Cloud Run, avec un front léger sur Vercel. Objectif : accélérer la mise en production d’usages IA fiables (auditables, observables, isolés par agent).

Voir la démo technique   /api/health • /gateway/health

Ce que nous faisons

  • Une gateway FastAPI qui proxifie des agents isolés (Cloud Run) : /agents/<nom>.
  • Des agents unitaires (ex. “echo”, “gemini”) packagés en conteneurs, déployés indépendamment.
  • Un front minimal (Vercel Functions + static) pour tester et intégrer rapidement.
  • Observabilité (logs Cloud Run + Vercel), sécurité par service account & env per agent.

Problèmes que nous résolvons

  • Éviter les conflits entre agents et les “hallucinations” croisées via isolation par service.
  • Réduire le time-to-prod : build & deploy en 1 commande (Cloud Build).
  • Offrir une surface API stable pour des intégrations web & internes.

Produits (état actuel)

  • Agent Gateway (Cloud Run) — routes /agents/echo, /agents/gemini.
  • Agent “Echo” — service de test pour valider la chaîne réseau.
  • Agent “Gemini” — génération texte (Vertex AI, gemini-1.5-flash).
